# TX PR Auditor Technical Design

## Document Control

| Field | Value |
|---|---|
| Skill | `tx-pr-auditor` |
| Status | Revamped create-pr-cd output validator |
| Runtime entrypoint | `scripts/audit_final_po.py` |
| Primary output | `output/PR_Audit_Result.xlsx` |
| Summary output | `output/PR_Audit_Result.summary.json` |
| Design/log location | repository-level `./prompts` |

## Purpose

TX PR Auditor validates submitted TX PR/PO lines in `Final PO.xlsx` against ECC workbooks generated by `create-pr-cd`.

The auditor does not read `EPMS.xlsx` or `pr_model.xlsx`. Those are upstream generation inputs owned by `create-pr-cd`.

`--expected-ecc` is required and can be repeated. Each value can be an ECC workbook or a directory containing generated ECC workbooks.

`--final-po-max-rows` can limit Final PO data rows for smoke tests against very large exports.

Optional annotated ECC output:

```bash
python scripts/audit_final_po.py \
  --final-po "input/Final PO.xlsx" \
  --final-po-sheet "Sheet1" \
  --final-po-header-row 2 \
  --expected-ecc "../create-pr-cd/output" \
  --output "output/PR_Audit_Result.xlsx" \
  --summary-json "output/PR_Audit_Result.full.summary.json" \
  --annotate-ecc-output \
  --annotated-ecc-output-root "output"
```

## Pipeline

```text
Workbook Reader
Field Mapper
Canonical Builder
Expected ECC Matcher
Audit Engine
Duplicate Resolver
Report Writer
```

When annotation is enabled, the pipeline also runs:

```text
Annotated ECC Writer
```

## Matching Model

Generated ECC rows are expected entitlement. Final PO rows are submitted claims.

Primary matching:

- Final PO physical site code to ECC `Site ID*`
- Final PO DU/logical site code to ECC `Delivery Unit Code*`
- Final PO item code to ECC `PBOM Code*`
- Final PO subcontractor to ECC `Subcontractor*`

Scope is inferred from ECC file names when possible:

- `TSS PR`
- `TI PR`
- `Planning PR`
- `Operation PR`

## Classification

| Classification | Reason code | Condition |
|---|---|---|
| Normal | `NORMAL_FULL` | Submitted row matches generated ECC entitlement and quantity remains. |
| Abnormal - Invalid PO | `INVALID_NOT_IN_CREATE_PR_CD_OUTPUT` | Submitted site or DU has no generated ECC entitlement. |
| Abnormal - Invalid PO | `INVALID_SUBCON_CHANGED` | Submitted subcontractor differs from generated ECC subcontractor. |
| Abnormal - Wrong PO | `WRONG_LINE_ITEM_MAPPING` | Site/DU exists but submitted item code is not expected. |
| Abnormal - Duplicate PO | `DUPLICATE_FULL_QUANTITY` | Entire submitted quantity exceeds remaining ECC entitlement. |
| Abnormal - Duplicate PO | `DUPLICATE_PARTIAL_QUANTITY` | Part of submitted quantity exceeds remaining ECC entitlement. |

## Output Columns

The writer preserves original Final PO columns and appends:

- Source Row
- Scope
- Audit Result
- Reason Code
- Expected Item
- Expected Quantity
- Expected Subcontractor
- Normal Quantity
- Duplicate Quantity
- Expected ECC Evidence
- Matched ECC Evidence
- Explanation

## Annotated ECC Output

When enabled, the auditor copies every generated ECC workbook into `output/<timestamp>/` and appends compact status columns to each copied `details` sheet.

The source create-pr-cd files are never modified. Rows with no exact Final PO item match are labeled `NOT_IN_FINAL_PO`.
