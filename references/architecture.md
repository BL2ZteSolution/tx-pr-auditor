# TX PR Auditor Architecture

## Purpose

TX PR Auditor validates submitted TX PR/PO records in `Final PO.xlsx` against the ECC workbooks generated by `create-pr-cd`.

Classifications:

- Normal
- Abnormal - Invalid PO
- Abnormal - Wrong PO
- Abnormal - Duplicate PO

## Product Boundary

`create-pr-cd` responsibilities:

- Interpret site/EPMS data.
- Resolve PR Model and contract information.
- Generate ECC output workbooks.
- Emit review-required generation evidence.

`tx-pr-auditor` responsibilities:

- Read Final PO rows.
- Read generated ECC rows.
- Normalize both datasets.
- Compare submitted PO rows to generated ECC entitlement.
- Resolve duplicates.
- Write audit results.

The auditor must not read `EPMS.xlsx` or `pr_model.xlsx`.

## Pipeline Model

Use a centralized batch pipeline:

```text
Workbook Reader
Field Mapper
Canonical Builder
Expected ECC Matcher
Audit Engine
Duplicate Resolver
Report Writer
```

Only the pipeline orchestrator controls execution.

## Immutable Datasets

Each stage emits a new typed dataset and does not mutate previous outputs:

```text
Raw Dataset
Canonical Dataset
Expected Match Dataset
Audit Dataset
Duplicate Dataset
Output Dataset
```

## Stage Responsibilities

Workbook Reader:

- Load explicit workbook paths.
- Read Final PO and generated ECC rows.
- Emit raw rows and workbook metadata.

Field Mapper:

- Convert workbook headers into canonical fields.
- Preserve raw row data and source workbook evidence.

Canonical Builder:

- Normalize site codes, DU codes, item codes, subcontractors, quantities, and dispatch sort keys.
- Infer scope from generated ECC file names when possible.

Expected ECC Matcher:

- Match Final PO rows to ECC entitlement by site code or DU.
- Narrow to exact item-code matches.
- Preserve generated ECC evidence.

Audit Engine:

- Classify missing site/DU entitlement as Invalid.
- Classify subcontractor mismatch as Invalid.
- Classify item-code mismatch within an entitled site as Wrong.
- Mark otherwise-valid rows as pending for quantity resolution.

Duplicate Resolver:

- Process only otherwise-valid claims.
- Consume generated ECC expected quantity in Dispatch Date, Request Number, Dispatch Order Number, PO Line Number order.
- Ensure Wrong and Invalid rows do not consume quantity.

Report Writer:

- Generate `PR_Audit_Result.xlsx`.
- Preserve original Final PO columns.
- Append audit result columns and evidence fields.
- Optionally write JSON summary.
